# SPIRITS




---

UNION в SQL
---

1. Что такое UNION? (Определение)

UNION — это оператор языка SQL, который объединяет результаты двух или более запросов SELECT в один результирующий набор строк.

Простыми словами: это как если бы вы склеили две таблицы сверху вниз (вертикальное объединение), а не слева направо (как JOIN).

---

2. Зачем нужен UNION? (Сферы применения)

· Собрать данные из разных таблиц с одинаковой структурой (например, продажи за 2024 и 2025 год)
· Объединить данные из разных баз данных или разных серверов
· Создать единый справочник (например, всех сотрудников компании из разных отделов)
· Подготовить отчеты, где нужно свести информацию из нескольких источников
· Выгрузить архивные данные вместе с текущими

---

3. Главные правила UNION (обязательно запомнить!)

Чтобы использовать UNION, у всех запросов должны быть:

Правило Пояснение
Одинаковое количество столбцов В первом SELECT 3 столбца → во втором тоже 3
Одинаковые типы данных Если в первом столбце числа, то и во втором числа
Одинаковый порядок столбцов Сначала имя, потом возраст → и в других запросах так же

Если эти правила нарушены — SQL выдаст ошибку.

---

4. Базовый пример

Есть две таблицы:

Таблица сотрудники_офис1  Таблица сотрудники_офис2 
имя отдел имя отдел
Анна IT Олег HR
Борис IT Мария IT

Запрос:

```sql
SELECT имя, отдел FROM сотрудники_офис1
UNION
SELECT имя, отдел FROM сотрудники_офис2;
```

Результат:

имя отдел
Анна IT
Борис IT
Олег HR
Мария IT

5. UNION vs UNION ALL — КЛЮЧЕВОЕ отличие!

Характеристика UNION UNION ALL
Удаляет дубликаты ДА НЕТ
Скорость работы Медленнее Быстрее
Ресурсы (память/CPU) Тратит на поиск дублей Не тратит
Когда использовать Нужны уникальные строки Важны все строки (включая повторы)

Пример с дубликатами:

Таблицы:

продажи_январь  продажи_февраль
клиент сумма клиент
Иванов 1000 Иванов
Петров 2000 Сидоров

UNION:

```sql
SELECT клиент FROM продажи_январь
UNION
SELECT клиент FROM продажи_февраль;
```

Результат: Иванов, Петров, Сидоров (3 строки)

UNION ALL:

```sql
SELECT клиент FROM продажи_январь
UNION ALL
SELECT клиент FROM продажи_февраль;
```

Результат: Иванов, Петров, Иванов, Сидоров (4 строки)

6. UNION с ORDER BY

Порядок сортировки применяется ко всему результату.

```sql
SELECT имя, возраст FROM ученики_9А
UNION
SELECT имя, возраст FROM ученики_9Б
ORDER BY возраст DESC;
```

Результат будет отсортирован по возрасту (от старшего к младшему) для ВСЕХ учеников из обоих классов.

Важно! ORDER BY пишется в самом конце, после всех SELECT.

---

7. UNION с разными названиями столбцов

Названия столбцов в итоговой таблице берутся из ПЕРВОГО запроса.

```sql
SELECT имя AS "ФИО", возраст AS "Лет" FROM таблица1
UNION
SELECT full_name, age FROM таблица2;
```

В результате столбцы будут называться ФИО и Лет.

8. частые ошибки

Ошибка Что делать
Разное количество столбцов Добавить пустой столбец: SELECT имя, NULL FROM ...
Разные типы данных Преобразовать типы: CAST(возраст AS VARCHAR)
ORDER BY внутри каждого SELECT Убрать, оставить только в конце
Скобки забыл Когда три SELECT, лучше обернуть каждый в скобки: (SELECT ...) UNION (SELECT ...)

Как исправить разное количество столбцов:

```sql
SELECT имя, возраст, 'офис1' AS источник FROM офис1
UNION
SELECT имя, возраст, NULL FROM офис2;  -- добавляем фиктивный столбец
```

---

9. UNION с тремя запросами

Можно объединять сколько угодно запросов:

```sql
SELECT город, сумма FROM продажи_2022
UNION ALL
SELECT город, сумма FROM продажи_2023
UNION ALL
SELECT город, сумма FROM продажи_2024;
```

10. UNION vs JOIN — в чем разница? (важно!)

 UNION JOIN
Как объединяет? Вертикально (сверху вниз) Горизонтально (слева направо)
Что добавляет? Новые строки Новые столбцы
Условие Не нужно Нужно (ON или USING)
Пример Список учеников 9А + список 9Б Таблица учеников + их оценки по предметам

Наглядная схема:

```
UNION:           JOIN:
[Табл.А]        [Табл.А]  [Табл.Б]
[Табл.Б]        строка1 + строка1
                 строка2 + строка2
```

11. Сложный пример (для продвинутых)

Задача: Показать всех клиентов, которые совершили покупки в январе или феврале, без дубликатов, отсортировать по имени.

```sql
SELECT 
    клиент_id, 
    имя, 
    'январь' AS месяц
FROM заказы_январь
WHERE сумма > 500

UNION

SELECT 
    клиент_id, 
    имя, 
    'февраль' AS месяц
FROM заказы_февраль
WHERE сумма > 500

ORDER BY имя;
```

---



1. UNION = вертикальная склейка, удаляет повторы
2. UNION ALL = вертикальная склейка, оставляет всё как есть (быстрее)
3. Количество, типы и порядок столбцов должны совпадать во всех SELECT

---

Если в результате UNION нужно показать, из какой таблицы пришла строка, добавьте текстовую метку:

```sql
SELECT имя, 'офис1' AS филиал FROM сотрудники_офис1
UNION
SELECT имя, 'офис2' AS филиал FROM сотрудники_офис2;
```

Результат:

имя филиал
Анна офис1
Борис офис1
Олег офис2
